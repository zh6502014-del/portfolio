DESIGN.md 是讓 AI agent 讀取並生成一致 UI 的純文字設計系統文件，由 Google Stitch 提出，awesome-design-md 收錄了眾多品牌的範例
什麼是 DESIGN.md？
概念說明
定義放在專案根目錄的一個 Markdown 檔案，描述你的設計系統。AI agent 讀取後能生成風格一致的 UI。不需要 Figma exports、JSON schema 或特殊工具。
用法把 DESIGN.md 貼進對話，告訴 AI「用這個設計系統生成元件」。後續所有請求都會自動遵循設計語言。
優點設計決策文字化 → 版本控管、團隊共享、AI 可讀；比截圖或 Figma link 更穩定可靠。

本作品集 Do / Don't
✅ Do❌ Don't
使用 glass-el 四層 box-shadow 完整套用玻璃效果只加 backdrop-filter 不加 border 和 box-shadow
主題切換用 data-theme 屬性 + CSS 變數，不用 class toggle直接 hardcode 顏色值（改用 CSS variable）
Nav Pill 用 CSS variable（--pill-x / --pill-w）控制位置，JS 只改變數用 JS 直接設 left / width style（破壞 transition）
圖片封面用 16:9 aspect-ratio + object-fit:cover圖片用固定 px 高度（RWD 會破版）
Modal 開關用 .open class + opacity + pointer-events，保留 DOM用 display:none/block 切換（失去 transition）
背景光暈球用 position:fixed + blur(80px) + z-index:0把光暈球放進 content flow（影響 layout）
Supabase 前台用原生 fetch（sbFetch），不用 SDK前台引入 @supabase/supabase-js（觸發 BroadcastChannel DataCloneError）
富文字 content 用 q.root.innerHTML 存取，setEditorHTML() 載入直接用 q.setText() 或 q.setContents()（遺失 HTML 格式）

# DESIGN.md — 林艾奭 UI/UX 作品集

## 1. Brand Identity
- **Name**: 林艾奭作品集
- **Keywords**: Liquid Glass, 沉穩極簡, 動態光感, 台灣金融科技
- **Aesthetic**: Glass morphism 為核心，背景光暈球製造深度，Nav pill 滑動回饋互動感；整體克制、不過度裝飾。

## 2. Color Palette
| Role | Light | Dark |
|------|-------|------|
| Background | #ffffff | #0e0d0c |
| Background-2 | #f4f3f0 | #1c1a18 |
| Background-3 | #eceae5 | #252320 |
| Text | #111110 | #f0ede8 |
| Text-secondary | #4a4945 | #b8b5b0 |
| Text-tertiary | #88867f | #706d68 |
| Border | rgba(0,0,0,.13) | rgba(255,255,255,.10) |
| Accent | #111110 | #f0ede8 |
| Glass-bg | rgba(255,255,255,.6) | rgba(255,255,255,.07) |

## 3. Typography
- **Font**: Noto Sans TC (300/400/500/600/700) → system-ui → sans-serif
- **Scale**: Hero 48px / Section 32px / Card 22px / Body 16-20px / Caption 13-14px
- **Line-height**: 1.6 (UI) / 1.8-1.9 (body text)
- **Letter-spacing**: -0.02em (hero titles only)

## 4. Spacing & Layout
- Max-width: 1100px
- Wrapper padding: 0 40px (mobile: 0 18px)
- Border-radius: sm 6px / md 8px / lg 12px
- Section gap: 80px (mobile: 48px)

## 5. Components
- **Glass card**: backdrop-filter blur(12px) saturate(160%) + 4-layer box-shadow
- **Nav pill**: CSS var --pill-x/--pill-w, JS only updates vars, CSS handles transition 400ms
- **Modal**: .open class + opacity/pointer-events toggle, shared DOM for projects & notes
- **Tag**: glass-el style, border-radius 100px
- **Button**: accent bg, border-radius 8px, hover opacity .85

## 6. Patterns & Motion
- Glass hover: gst 360ms cubic-bezier(1,0,.4,1) — horizontal stretch scale(1.08)
- Pill slide: translate 400ms + width 400ms cubic-bezier(1,0,.4,1)
- Theme transition: 300ms, class .theme-transitioning disables pointer-events
- Scroll: passive listener, updateNav() determines active section

## 7. Imagery & Icons
- Cover images: 16:9, border-radius 8px, object-fit:cover
- Nav/skill icons: emoji
- Action icons: SVG line style, stroke-width 1.5-2px
- Lightbox: full-screen overlay, pinch-zoom on mobile, swipe to navigate

## 8. Voice & Tone
- 語氣：中性、專業、不過度謙遜
- 中文優先；英文專有名詞保留原文（UI/UX、Design System）
- 標題簡潔，body text 完整說明

## 9. Do / Don't
✅ DO
- Use complete 4-layer glass box-shadow
- Control Nav pill position via CSS variables only
- Use sbFetch() (native fetch) for frontend Supabase calls
- Use q.root.innerHTML to read/write Quill editor content

❌ DON'T
- Hardcode color values (use CSS variables)
- Use display:none/block for modal toggle (breaks transitions)
- Import @supabase/supabase-js in frontend (BroadcastChannel error)
- Use q.setText() or q.setContents() for loading rich HTML
